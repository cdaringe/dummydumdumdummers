# Scenario 43: Self-Hosting with Docker

> The project SHALL be excellent to self host. Hobbyists SHOULD feel easy launching a container with docker-in-docker.

## Implementation

### Docker Deployment

- `Dockerfile` — multi-stage build, minimal runtime image (Node 22 Alpine)
- `docker-compose.yml` — one-command deployment with volume mounts
- `DOCKER.md` — step-by-step Docker documentation

### Ease of Use

```bash
docker build -t thingfactory .
docker run --rm thingfactory run basic
```

For persistent deployment with database:
```bash
docker-compose up
```

### Documentation

`docs/HOSTING_SERVICE.md` covers:
- Docker deployment, environment variables, volume mounts
- Worker configuration, reverse proxy setup
- Production monitoring and health checks
