# Scenario 3: Diataxis-Compliant Documentation

> The project shall offer documentation adherent to Diataxis, mainly "how to" and "reference". We value conciseness and linking to other resources over long-form documentation.

## Implementation

Six documentation guides follow the Diataxis framework, focusing on "how to" and "reference" categories. Each guide includes a Table of Contents.

### Documentation Structure

| Guide | Diataxis Type | Description |
|---|---|---|
| `docs/USER_GUIDE.md` | Reference | Architecture overview, key concepts |
| `docs/GETTING_STARTED.md` | How-to | Install and run first pipeline |
| `docs/RUNNING_PIPELINES.md` | How-to | CLI modes, Docker execution, artifacts |
| `docs/WEB_GUI_GUIDE.md` | Reference | All GUI pages and features |
| `docs/HOSTING_SERVICE.md` | How-to | Production deployment, configuration |
| `docs/TROUBLESHOOTING.md` | Reference | 50+ common issues with solutions |
