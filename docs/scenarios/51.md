# Scenario 51: Deployment Guide

**Specification:** The project SHALL provide a deployment guide.

## Evidence

A complete deployment guide is hosted in two documents:

### Primary: `docs/HOSTING_SERVICE.md`

Covers all deployment paths with a Table of Contents:

- **Docker Deployment** — multi-stage build, container run, persistent storage,
  Docker-in-Docker socket mounting
- **Manual Node.js Deployment** — build steps, production start command
- **Configuration** — environment variables (`PORT`, `NODE_ENV`,
  `DATABASE_PATH`)
- **Database** — SQLite auto-migration, backup procedure
- **Reverse Proxy** — nginx config with SSE header support
- **Kubernetes** — using `kubernetes_runner.step()` with RBAC requirements
- **Self-Hosting Tips** — single-container hobbyist setup, workers, isolation,
  secrets, monitoring

### Supplementary: `DOCKER.md`

Docker-specific guide covering:

- CLI container (single-stage Gleam → JS build)
- Web GUI container (`web/Dockerfile`)
- `docker-compose.yml` services (`web` on port 3000, `cli`)
- Configuration table
- Self-hosting with Docker socket mounting for pipeline isolation

### Supporting Files

| File                      | Purpose                                               |
| ------------------------- | ----------------------------------------------------- |
| `Dockerfile`              | Multi-stage CLI image (node:22-alpine + Gleam 1.13.0) |
| `web/Dockerfile`          | Web GUI image                                         |
| `docker-compose.yml`      | Full-stack compose with volume mounts and socket      |
| `DOCKER.md`               | Docker quick-start and reference                      |
| `docs/HOSTING_SERVICE.md` | Complete production deployment guide                  |

The deployment guide covers Docker, manual Node.js, Kubernetes, and self-hosting
scenarios, satisfying the requirement.
