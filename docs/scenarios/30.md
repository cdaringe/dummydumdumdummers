# Scenario 30: CLI for Local Development

> The system SHALL offer a CLI to run pipelines locally, exactly as they would
> run in production.

## Implementation

The CLI uses the same executor and parallel_executor as production. Pipelines
run identically whether invoked locally or deployed.

### CLI Entry Points

- `gleam run -m thingfactory/cli -- run <pipeline>` — local execution
- `docker run thingfactory run <pipeline>` — Docker execution (same code path)

### Same Execution Path

Both CLI and production use:

- `executor.execute()` for sequential pipelines
- `parallel_executor.execute_parallel()` for DAG pipelines
- `command_runner.step()` for shell commands
