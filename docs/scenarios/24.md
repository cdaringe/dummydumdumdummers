# Scenario 24: Resource Consumption from Pipeline Nodes

> Pipelines SHOULD be able to consume resources (artifacts, APIs, databases,
> files) from other nodes with high ease.

## Implementation

Three mechanisms enable inter-step resource sharing:

1. **Artifact Store** — `Context.artifact_store` for keyed data sharing between
   steps
2. **Message Bus** — `Context.message_store` for pub-sub event coordination
3. **Dependency Injection** — `Context.deps` for injected external resources
   (APIs, DB connections)

Steps access all three via the `Context` parameter. `add_step_with_ctx` enables
steps to update artifacts and messages.
