# Scenario 20: Extensible Task Runners

> Pipeline task runners SHALL be extensible to support custom use cases.

## Implementation

Users can create custom step factories by defining functions that return pipeline-compatible step functions. The `custom_command_step` factory in examples demonstrates this pattern.

### Custom Runner Pattern (`src/thingfactory/examples.gleam:383-467`)

```gleam
pub fn custom_command_step(cmd: CommandStep)
  -> fn(Context, Dynamic) -> Result(Dynamic, StepError) {
  fn(_ctx, _input) { ... }
}
```

Any `fn(Context, Dynamic) -> Result(Dynamic, StepError)` can be a step. This includes:
- Shell commands (`command_runner.step()`)
- Kubernetes Jobs (`kubernetes_runner.step()`)
- Custom factories wrapping HTTP calls, database queries, etc.
