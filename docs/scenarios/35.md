# Scenario 35: Proper CLI Parser

> The CLI SHALL be designed to be published and used by customers. A real,
> proper CLI parser should be used.

## Implementation

CLI command parsing is implemented with `clip` and now exposes explicit
customer-facing subcommands for key jobs:

- `run` - execute pipeline with compact/verbose progress controls
- `inspect` - execute pipeline then open interactive REPL for result drill-down
- `results` - execute pipeline and print a detailed step-by-step result report
- `artifacts` - execute pipeline and extract produced artifacts to
  `-o/--output-dir`
- `list` - print supported invocation patterns

### CLI Structure (`src/thingfactory/cli.gleam`)

- **Real subcommand parser**: `clip.subcommands(...)` for `run`, `inspect`,
  `results`, `artifacts`, `list`
- **Required options where needed**: `artifacts` requires `-o/--output-dir`
- **Runtime pipeline loading**: each execution command supports `-f/--file` and
  module/function selectors
- **Result/report paths**: parser routes to execution + summary/report/artifact
  flows without stringly dispatch logic
