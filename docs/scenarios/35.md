# Scenario 35: Proper CLI Parser

> The CLI SHALL be designed to be published and used by customers. A real, proper CLI parser should be used.

## Implementation

Uses the `clip` library for proper argument parsing with subcommands, flags, options, and auto-generated help.

### CLI Structure (`src/thingfactory/cli.gleam`)

- **Subcommands**: `run <pipeline>`, `list`
- **Flags**: `-c/--compact` (compact output), `-i/--interactive` (interactive mode)
- **Options**: `-o/--output-dir <dir>` (artifact extraction)
- **Auto-generated help**: `clip.help(help.simple(...))` on each subcommand
- **Error handling**: unknown commands, missing arguments, invalid pipelines
