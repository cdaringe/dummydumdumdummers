# Scenario 41: Async, Parallel, and Accumulating Examples

> The project SHALL host examples that have asynchronous steps, parallel steps, and steps that accumulate values and pass them to later steps. Each step should run on a different node (such as different docker container) to demonstrate distributed execution.

## Implementation

Two dedicated distributed examples were added:

- `distributed_parallel_pipeline()`
- `distributed_accumulation_pipeline()`

Both examples execute each step via `kubernetes_runner.step(...)` with distinct job names (`s41-...-node-a/b/c/d`), so every step is scheduled as a separate Kubernetes Job pod/node.

### Distributed Async + Parallel

- `seed -> (async_left || async_right) -> merge` DAG in `distributed_parallel_pipeline()`
- Parallel fan-out/fan-in proves asynchronous step opportunities while running each step on separate nodes/pods.

### Distributed Accumulation + Pass-through

- `distributed_accumulation_pipeline()` runs four Kubernetes-backed steps on distinct nodes.
- `node_b_append` and `node_c_append` append new remote step outputs onto prior accumulated input and pass it to later steps.
- `node_d_publish` receives the accumulated value and publishes it on a different node.
