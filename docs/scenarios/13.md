# Scenario 13: Streaming stdio to CLI

> Pipeline tasks SHALL stream stdio to the user when executed locally using the CLI.

## Implementation

The CLI provides three output modes with real-time streaming via progress callbacks:

### Output Modes (`src/thingfactory/cli.gleam:94-101`)

- **Verbose** (default) — `verbose_progress()` prints step start/finish with status and duration in real time
- **Compact** (`-c`) — `compact_progress()` prints `[N/M] step_name ✓ duration` per step
- **Interactive** (`-i`) — REPL mode with `list`, `step <N>`, `stats`, `artifacts` commands

### Progress Callback System

`executor.execute_with_progress()` and `parallel_executor.execute_parallel_with_progress()` accept a `fn(StepEvent) -> Nil` callback. Events are emitted:
- `StepStarting(name, index, total)` — before execution
- `StepFinished(name, index, total, status, duration_ms)` — after execution
