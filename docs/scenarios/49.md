# Scenario 49: CLI Integration Tests

> The CLI SHALL have integration tests that verify all major features of the CLI work, including using the docker based execution.

## Completed

Added `test/cli_integration_test.gleam` with **13 new tests** covering all major CLI features, and exposed `resolve_isolation_mode` as a public function in `src/thingfactory/cli.gleam` for testability.

## Evidence

### New test file: `test/cli_integration_test.gleam`

**Isolation mode resolution (5 tests)** – pure-function, zero-overhead:

| Test | Verifies |
|---|---|
| `isolation_defaults_to_docker_test` | No `--isolator` flag → `DockerIsolation` with default Gleam image |
| `isolation_explicit_local_test` | `--isolator local` → `LocalIsolation` |
| `isolation_docker_with_custom_image_test` | `--isolator docker --docker-image alpine:3` → `DockerIsolation("alpine:3")` |
| `isolation_interactive_forces_local_test` | `-i` (interactive) forces `LocalIsolation` regardless of flag |
| `isolation_invalid_isolator_returns_error_test` | Unknown isolator name returns `Error` containing "Invalid isolator" |

**Subprocess integration tests (8 tests)** – invoke the compiled CLI binary end-to-end via `gleam run -m thingfactory/cli`:

| Test | Command | Verifies |
|---|---|---|
| `subprocess_list_command_test` | `list` | stdout non-empty, contains "thingfactory run" |
| `subprocess_run_local_verbose_test` | `run --isolator local` | stdout contains "Result:" |
| `subprocess_run_local_compact_test` | `run -c --isolator local` | stdout contains "steps" |
| `subprocess_run_from_file_test` | `run --isolator local -f src/... basic_pipeline` | stdout contains "Result:" |
| `subprocess_results_command_test` | `results` | stdout contains "Pipeline Results" and "SUCCESS" |
| `subprocess_artifacts_command_test` | `artifacts -o /tmp/...` | stdout mentions artifact extraction |
| `subprocess_invalid_ref_emits_error_test` | `run --isolator local not_valid` | stdout contains "Error" |
| `subprocess_docker_isolation_invoked_test` | `run --isolator docker` | stdout does NOT contain "Invalid pipeline reference" or "Invalid isolator" — confirming the CLI routes to the docker execution branch |

### Changed: `src/thingfactory/cli.gleam`

`resolve_isolation_mode` changed from `fn` to `pub fn` (line 753) to enable direct unit testing of the docker/local routing logic without spawning a subprocess.
