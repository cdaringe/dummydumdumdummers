# Scenario 50: Pipeline Graph With Edges

> The GUI SHALL show the graph of a pipeline with edges connecting every step.

## Implementation

- **DAG component** — `web/components/pipeline/PipelineDag.tsx` renders a React
  Flow canvas with a hidden `[data-testid="pipeline-dag-edges"]` div listing
  every edge as `<span data-testid="edge-{source}->{target}">`, enabling E2E
  assertions on edge wiring without depending on React Flow internals.
- **Graph builder** — `web/lib/dag.ts:buildDagGraph` converts every `depends_on`
  relation into a React Flow edge (`id: "<dep>-><step>"`, `source`, `target`),
  so every declared dependency becomes a drawn connection.
- **Pipeline detail page** — `web/app/pipelines/[name]/[version]/page.tsx` calls
  `buildDagGraph(steps, traceStatuses)` and passes the result to
  `<PipelineDag>`.
- **DB isolation fix** — `web/lib/db.ts` calls `seedFixturesIfEmpty(rawDb)` at
  module init when `DATABASE_PATH=:memory:`. Next.js Turbopack creates separate
  module instances for API routes vs. server components; without auto-seeding,
  page-level DB instances would start empty and 404.
- **Seed fixtures** — `web/lib/seed-fixtures.ts` provides 24 realistic
  pipelines. `typescript_build` has 6 steps (`checkout`, `install-deps`, `lint`,
  `compile`, `unit_tests`, `package`) producing 6 edges. `parallel_build` (5
  steps) demonstrates fan-out/fan-in DAG topology.
- **Tests** — `web/e2e/pipeline-detail.spec.ts` (11 tests): asserts DAG canvas
  visible, exactly 6 edges for `typescript_build`, concrete edge
  `install-deps->lint` is attached, step names `checkout`/`unit_tests` in table,
  `parallel_build` shows 5 steps. All 60 E2E tests pass.

This fulfills the requirement: the pipeline detail GUI renders a React Flow DAG
where each step is connected by edges reflecting its declared dependencies,
verified end-to-end via E2E coverage.
