# Scenario 38: CLI Defaults To Docker Isolation

> The CLI SHALL by default run pipelines in isolation. The isolation mechanism
> should be pluggable, but default to docker containers.

## Completed

`thingfactory run` defaults to Docker isolation and supports explicit backend
selection:

- `--isolator docker|local` (pluggable selector)
- `--docker-image <image>` (Docker backend config; default
  `ghcr.io/gleam-lang/gleam:v1.14.0-erlang`)

If `--isolator` is omitted, `docker` is used.

## How It Works

- `run` command arguments are parsed with isolator settings.
- In Docker mode, CLI re-executes itself inside a container and forces the inner
  command to `--isolator local` to prevent recursion.
- The container mounts the current workspace (`-v <cwd>:/workspace`) and runs:
  `gleam build --target erlang --warnings-as-errors && gleam run -m thingfactory/cli -- ...`
- Interactive mode (`-i`) automatically falls back to local isolation since it
  requires direct stdin/stdout.
