# Scenario 38: Default Docker Isolation

> The CLI SHALL by default run pipelines in isolation. The isolation mechanism should be pluggable, but default to docker containers.

## Implementation

### Pluggable Backends (`src/thingfactory/runner_host.gleam:40-46`)

```gleam
pub type RunnerBackend {
  Local        // In-process execution (default for development)
  Kubernetes(config: KubernetesConfig)  // K8s Jobs for production
}
```

### Docker Deployment

The Dockerfile packages the CLI for container execution. In production, `docker run thingfactory run <pipeline>` provides full isolation.

### Command Runner Isolation

`command_runner.step()` executes shell commands — when running inside Docker, this provides process-level isolation. The Kubernetes backend provides pod-level isolation.
