# Scenario 5: Testable Pipelines

> Pipelines SHALL be testable

## Implementation

Pipelines are testable via a mock injection system in `test_helpers.gleam`.
Steps can be replaced with mocks that return predefined results, enabling unit
testing without executing real side effects.

### Test Helpers API (`src/thingfactory/test_helpers.gleam`)

- `mock_step_success(name, output)` — mock returning `Ok(output)`
- `mock_step_error(name, error)` — mock returning `Error(error)`
- `mock_step_fn(name, fn)` — mock with custom logic
- `run_with_mocks(pipeline, mocks, input)` — execute pipeline with mocks
  substituted by step name

### How It Works

`run_with_mocks` replaces step functions by matching on step name in a `Dict`.
Non-mocked steps run normally. This is demonstrated in
`examples.gleam:mockable_pipeline()` and tested in
`test/test_helpers_test.gleam`.
