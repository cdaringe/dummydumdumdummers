# Scenario 26: Kubernetes Runner Backend

> The runner host SHALL allow kubernetes as a runner backend, but SHOULD be able to run on a single machine.

## Implementation

### Kubernetes Backend (`src/thingfactory/kubernetes_runner.gleam`)

- `KubernetesConfig` — namespace, image, service account, kubeconfig, resource limits
- `step(config, job_name, command)` — creates pipeline steps that run as K8s Jobs
- `build_job_yaml()` — generates Job manifests
- kubectl integration: `apply_manifest()`, `wait_for_job()`, `get_logs()`, `delete_job()`

### Single Machine (Default)

- `runner_host.new()` defaults to `Local` backend
- No Kubernetes required for development or simple deployments

### Runner Host Backend Selection (`src/thingfactory/runner_host.gleam`)

- `RunnerBackend` type: `Local | Kubernetes(config)`
- `with_kubernetes(config, workers)` — create K8s-backed host
- `set_backend(host, backend)` — switch backend
