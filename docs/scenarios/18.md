# Scenario 18: GUI Pipeline State Visibility

> Pipelines states SHALL be visible through a GUI

## Implementation

The Next.js 15 web GUI provides pipeline state visibility across 5 pages:

- **Dashboard** (`web/app/page.tsx`) — total pipelines, runs, success/failure counts, recent runs
- **Pipelines** (`web/app/pipelines/page.tsx`) — all registered pipelines with name, version, description
- **Pipeline Detail** (`web/app/pipelines/[name]/[version]/page.tsx`) — DAG visualization via React Flow, execution history
- **Runs** (`web/app/runs/page.tsx`) — paginated run history with status filters
- **Run Detail** (`web/app/runs/[runId]/page.tsx`) — step traces, Gantt timeline, logs, artifacts

### API Endpoints

- `GET /api/runs` — list runs with status/pipeline filters
- `GET /api/runs/[runId]` — run detail with step traces
- `GET /api/runs/[runId]/stream` — real-time event streaming
