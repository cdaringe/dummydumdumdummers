# Scenario 11: Inter-Step Broadcasting and Coordination

> Pipeline tasks SHALL be able to broadcast messages to one another or coordinate if the user desires

## Implementation

A pub-sub message bus embedded in `Context` enables steps to publish and subscribe to typed messages on named topics.

### Message Store (`src/thingfactory/message_store.gleam`)

- `publish(store, topic, payload)` — broadcast a `Dynamic` payload on a topic
- `get_messages(store, topic)` — retrieve all messages for a topic
- `clear_topic(store, topic)` — remove all messages from a topic
- `topics(store)` — list active topics
- `has_messages(store, topic)` — check for messages

### Context Integration (`src/thingfactory/types.gleam:106-118`)

- `types.publish_message(ctx, topic, payload)` — publish and return updated context
- `types.get_messages(ctx, topic)` — retrieve messages from context
- Steps using `add_step_with_ctx` can update the message store
