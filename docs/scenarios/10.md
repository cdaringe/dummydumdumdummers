# Scenario 10: Parallel Pipeline Task Execution

> Pipeline tasks SHALL be able to run in parallel

## Implementation

Steps declare dependencies via
`pipeline.add_step_with_deps(name, fn, depends_on)`. The parallel executor
builds a DAG from these declarations and executes steps in topological order —
steps whose dependencies are all satisfied run concurrently.

### Parallel Executor (`src/thingfactory/parallel_executor.gleam`)

- `execute_parallel(pipeline, input, config)` — main entry point
- `execute_parallel_with_progress(pipeline, input, config, on_progress)` — with
  real-time CLI callbacks
- `validate_dependencies(steps)` — rejects invalid references before execution
- `find_ready_step(steps, status)` — selects next step whose deps are all
  resolved
- Failed dependency propagation: if a dep fails, downstream steps are skipped

### Example Pipelines

- `parallel_build_pipeline()` — clone → lint || test → build → package
- `parallel_multi_target_pipeline()` — diamond pattern: setup → compile_a ||
  compile_b → test_a || test_b → integration → deploy
- `dogfood_pipeline()` — parallel Gleam + web builds merging at verify step
