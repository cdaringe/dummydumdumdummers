# Scenario 27: Secrets Management

> The pipeline system SHALL provide secrets management for Pipelines.

## Implementation

### Secret Manager (`src/thingfactory/secret_manager.gleam`)

- `Secret` — opaque type that masks values in debug output
- `SecretStore` — opaque Dict-based storage
- `set(store, name, value)` / `get(store, name)` — CRUD operations
- `has_secret()`, `list_secrets()`, `delete()`, `merge()` — store management
- `mask_value(value)` — masks all but last 4 characters for logging
- `require_secret()` / `require_secrets()` — validation before execution
- `from_dict()` / `to_dict()` — serialization for environment variable loading

### Pipeline Integration

```gleam
pipeline.new("secure", "1.0.0")
|> pipeline.with_secrets(secrets)
|> pipeline.add_secret("api_key", "sk_live_xxx")
```

Secrets are injected into `Context.secret_store` during execution. Both
sequential and parallel executors propagate secrets.
