# Scenario 56 — GitHub Repository Integration in GUI

## Requirement

> The user SHALL be able to select a GitHub organization and repository to
> connect to in the GUI, and select an autopopulated branch from a dropdown to
> register for Pipelines.

## Implementation

### Database

Migration `web/db/migrations/0006_github_connections.sql` adds:

```sql
CREATE TABLE IF NOT EXISTS github_connections (
  id          TEXT PRIMARY KEY,
  token       TEXT NOT NULL,
  org         TEXT NOT NULL,
  repo        TEXT NOT NULL,
  branch      TEXT NOT NULL,
  pipeline_id TEXT,
  created_at  TEXT NOT NULL DEFAULT (strftime('%Y-%m-%dT%H:%M:%SZ', 'now'))
);
```

`web/lib/db.ts` adds `GitHubConnectionsTable` interface and includes it in
`ThingfactoryDB`. The in-memory schema (used in tests) is also updated.

### GitHub API Proxy Routes

Four server-side API routes proxy GitHub's REST API using the user's Personal
Access Token:

| Route                                        | Purpose                                                        |
| -------------------------------------------- | -------------------------------------------------------------- |
| `GET /api/github/orgs?token=`                | Returns user + all orgs the token belongs to                   |
| `GET /api/github/repos?token=&org=&type=`    | Lists repos for an org or user account                         |
| `GET /api/github/branches?token=&org=&repo=` | Lists all branches for a repo                                  |
| `GET /api/github/connections`                | Lists saved connections (token excluded)                       |
| `POST /api/github/connections`               | Saves a connection; optionally updates linked pipeline trigger |
| `DELETE /api/github/connections/[id]`        | Removes a connection                                           |

When a connection is saved with a `pipeline_id`, the pipeline's trigger is
automatically set to `{ GitHub: { repo: "org/repo", events: ["push"] } }`.

### UI — `/integrations` Page

`web/app/integrations/page.tsx` is a client component with cascading dropdowns:

1. **Token input** (`type="password"`) + "Load Organizations" button — calls
   `GET /api/github/orgs`
2. **Organization dropdown** — autopopulated from GitHub API; includes user
   account and all orgs
3. **Repository dropdown** — autopopulated when org is selected; calls
   `GET /api/github/repos`
4. **Branch dropdown** — autopopulated when repo is selected; pre-selects
   `default_branch`; calls `GET /api/github/branches`
5. **Pipeline selector** — optional; links the connection to an existing
   pipeline definition
6. **Register Connection** button — POSTs to `/api/github/connections`

Existing connections are listed in a table with org/repo, branch, linked
pipeline, timestamp, and a Remove button.
