# Scenario 19: Default Task Runners

> Pipeline task runners SHALL come with defaults for common use cases.

## Implementation

Three built-in runner backends cover common use cases:

- **Local** (`executor.gleam`) — sequential in-process execution, default for
  CLI
- **Parallel** (`parallel_executor.gleam`) — DAG-aware execution for multi-step
  pipelines
- **Command Runner** (`command_runner.gleam`) — shell command execution with
  `step()` factory

`runner_host.gleam` defaults to `Local` backend with one worker per CPU core.

### Evidence

| Artifact                                    | Description                                               |
| ------------------------------------------- | --------------------------------------------------------- |
| `src/thingfactory/executor.gleam`           | Sequential executor (default)                             |
| `src/thingfactory/parallel_executor.gleam`  | Parallel DAG executor                                     |
| `src/thingfactory/command_runner.gleam`     | Shell command runner with `step()` factory                |
