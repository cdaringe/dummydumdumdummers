# Scenario 52: THINGFACTORY_* Environment Variables

**Specification:** The service SHALL be configurable via ALL THINGFACTORY_*
environment variables, and the documentation SHALL provide a reference for all
of these. The variables MUST be correlated to a clean/crisp typed datamodel that
underpins the service.

## Implementation

### Typed Config Datamodel: `web/lib/config.ts`

A central `ServiceConfig` interface defines every configurable parameter:

```typescript
export interface ServiceConfig {
  databasePath: string;   // THINGFACTORY_DATABASE_PATH
  port: number;           // THINGFACTORY_PORT or PORT
  nodeEnv: "development" | "production" | "test";  // NODE_ENV
}

export function getConfig(): ServiceConfig { ... }
export const config = getConfig();
```

All `THINGFACTORY_*` variables are read with fallback to legacy names for
backwards compatibility:

- `THINGFACTORY_DATABASE_PATH` → falls back to `DATABASE_PATH` → defaults to
  `./db/thingfactory.db`
- `THINGFACTORY_PORT` → falls back to `PORT` → defaults to `3000`

### Variables Reference

| Variable                     | Default                | Description                                  |
| ---------------------------- | ---------------------- | -------------------------------------------- |
| `THINGFACTORY_DATABASE_PATH` | `./db/thingfactory.db` | SQLite database file path                    |
| `THINGFACTORY_PORT`          | `3000`                 | Web server port                              |
| `NODE_ENV`                   | `development`          | Node environment (`production` for deployed) |
