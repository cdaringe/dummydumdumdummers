# Scenario 17: Additional Trigger Types

> Pipelines SHOULD be able to run given other triggers

## Implementation

Beyond webhooks (Scenario 16), the system supports:

- `ManualTrigger` — explicit user-initiated execution via CLI or GUI
- Schedule-based triggers (Scenario 15) — time-based automatic execution
- `CustomMatcher(fn(WebhookEvent) -> Bool)` — arbitrary trigger logic via user-defined predicates

The `Trigger` type is extensible — new trigger variants can be added to the sum type.
