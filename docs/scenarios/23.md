# Scenario 23: Sensible CI/CD Defaults

> The GUI & system parameters SHOULD, when otherwise undefined, use behaviors and capabilities common in the CI/CD ecosystem.

## Implementation

Default behaviors follow CI/CD conventions:

- **30-minute default timeout** (`types.default_config()`) — standard CI timeout
- **Sequential execution** — steps run in order unless parallel deps declared
- **Error stops pipeline** — first failure aborts remaining steps (marked as skipped)
- **Trace on every step** — duration, status, errors recorded automatically
- **One worker per CPU core** — `runner_host.new()` auto-detects cores
