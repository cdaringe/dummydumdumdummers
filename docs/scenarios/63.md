# Scenario 63: THINGFACTORY_DATA_DIRNAME

**Specification:** A user deploying the server SHALL be able to specify a
`THINGFACTORY_DATA_DIRNAME=/some/path` and all writes shall be made to that
directory, including the SQLite database and any produced artifacts.

## Implementation

### `web/lib/config.ts` — `dataDirname` field

`ServiceConfig` gains a `dataDirname` field populated from
`THINGFACTORY_DATA_DIRNAME` (defaults to `./data`). The database path is derived
from it unless `THINGFACTORY_DATABASE_PATH` overrides:

```typescript
const dataDirname = process.env.THINGFACTORY_DATA_DIRNAME ?? "./data";
const databasePath = process.env.THINGFACTORY_DATABASE_PATH ??
  join(dataDirname, "db", "thingfactory.db");
```

On startup, the config module creates required subdirectories (`db/`, `logs/`,
`backups/`) inside `dataDirname` via `mkdirSync(..., { recursive: true })`.

### `docker-compose.yml` — Volume mount

The `web` service maps the host path `${THINGFACTORY_DATA_DIRNAME:-./data}` to
`/app/data` inside the container, and passes
`THINGFACTORY_DATA_DIRNAME=/app/data` to the process:

```yaml
volumes:
  - ${THINGFACTORY_DATA_DIRNAME:-./data}:/app/data
environment:
  - THINGFACTORY_DATA_DIRNAME=/app/data
```

### `web/Dockerfile` — Pre-create data dir

The runtime stage creates `/app/data` so the directory always exists on first
start even without a mounted volume:

```dockerfile
RUN mkdir -p /app/data
```

### Updated Variables Reference (`docs/DOCKER.md` / `docs/HOSTING_SERVICE.md`)

| Variable                     | Default     | Description                                 |
| ---------------------------- | ----------- | ------------------------------------------- |
| `THINGFACTORY_DATA_DIRNAME`  | `./data`    | Root for all writes (DB, logs, backups)     |
| `THINGFACTORY_DATABASE_PATH` | _(derived)_ | SQLite path; overrides data dirname default |

## Verification

- `web/lib/config.ts`: `dataDirname` and subdirectory creation confirmed.
- `docker-compose.yml`: volume and env var wiring confirmed.
- In-memory test databases (`":memory:"`) bypass directory creation (guard in
  `config.ts`).
