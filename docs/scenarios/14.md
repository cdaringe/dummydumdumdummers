# Scenario 14: Persisted stdio

> Pipeline tasks SHALL have stdio persisted and viewable after execution.

## Implementation

Step output is persisted in the database via the `step_traces` table, which includes a `log_output` column added by migration 0005. The web GUI renders persisted logs through `StepLogViewer.tsx`.

### Database Schema

- `web/db/migrations/0005_step_logs.sql` — adds `log_output TEXT` to `step_traces`
- `step_traces` table: id, run_id, step_name, status, duration_ms, error_msg, log_output, sequence, created_at

### GUI Viewing

`web/components/run/StepLogViewer.tsx` renders persisted logs with syntax highlighting and step status.

### Evidence

| Artifact | Description |
|---|---|
