# Scenario 60 — Root Pipeline File for Self-Building

## Requirement

> The project SHALL host a pipeline file in the root such that a deployed
> instance of the project can build itself.

## Implementation

### Pipeline File

`src/thingfactory/build.gleam` — 13-step parallel DAG pipeline covering the full
CI/CD lifecycle. Run via:

```bash
thingfactory run -f src/thingfactory/build.gleam build
```

### Pipeline Steps (all real `command_runner` invocations, zero mocks)

| Tier | Step               | Command                                                  | Depends On                                      |
| ---- | ------------------ | -------------------------------------------------------- | ----------------------------------------------- |
| 0    | `gleam_check`      | `gleam check`                                            | —                                               |
| 0    | `gleam_format`     | `gleam format --check`                                   | —                                               |
| 0    | `web_install`      | `npm --prefix web ci`                                    | —                                               |
| 1    | `gleam_test`       | `gleam test`                                             | check, format                                   |
| 1    | `web_lint`         | `npm --prefix web run lint`                              | web_lint                                        |
| 2    | `gleam_build_erl`  | `gleam build --target erlang`                            | gleam_test                                      |
| 2    | `gleam_build_js`   | `gleam build --target javascript`                        | gleam_test                                      |
| 2    | `web_build`        | `npm --prefix web run build`                             | web_lint                                        |
| 3    | `cli_shipment`     | `gleam export erlang-shipment`                           | gleam_build_erl                                 |
| 3    | `docker_build_cli` | `docker build -t thingfactory-cli .`                     | cli_shipment                                    |
| 3    | `docker_build_web` | `docker build -t thingfactory-web -f web/Dockerfile web` | web_build                                       |
| 4    | `hex_publish`      | `gleam hex publish --yes`                                | gleam_build_erl, gleam_build_js                 |
| 5    | `semantic_release` | `npx semantic-release --no-ci`                           | docker_build_cli, docker_build_web, hex_publish |

### Trigger

Pipeline configured with `BranchUpdate("main")` — automatically fires on pushes
to `main` via GitHub/Gitea webhooks (scenario 59).

### Test Coverage

`test/build_test.gleam` — 9 structure tests verify pipeline ID, step count (13),
trigger, timeout (600s), and every dependency edge across all 5 tiers. 285 total
tests pass.

### Key Files

- `src/thingfactory/build.gleam` — the pipeline definition
- `test/build_test.gleam` — structure tests (9 tests)
- `Dockerfile` — CLI Docker image (used by `docker_build_cli` step)
- `web/Dockerfile` — Web Docker image (used by `docker_build_web` step)
