# Scenario 39: User Documentation with TOC

> The site SHALL host user documentation on how to host the service, how to run
> Pipelines, and ALL other aspects. Every page SHALL have a TOC.

## Implementation

Six comprehensive documentation guides, each with Table of Contents:

1. `docs/USER_GUIDE.md` — Architecture overview, key concepts
2. `docs/GETTING_STARTED.md` — Installation and first pipeline
3. `docs/RUNNING_PIPELINES.md` — CLI modes, Docker, artifact extraction
4. `docs/WEB_GUI_GUIDE.md` — All GUI pages and features
5. `docs/HOSTING_SERVICE.md` — Docker deployment, configuration, production
   setup
6. `docs/TROUBLESHOOTING.md` — 50+ issues with solutions
