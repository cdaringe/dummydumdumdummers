# Scenario 45: Pipeline Statistics Dashboard

> The Pipeline GUI SHALL provide statistics about prior runs.

## Implementation

### Statistics Page (`web/app/stats/page.tsx`)

Dashboard showing:

- Overall stats: total runs, success rate, average duration
- Per-pipeline statistics: total runs, success/failure count, avg/min/max
  duration
- Trend data over time

### API Endpoint

`GET /api/stats` returns aggregated statistics from the pipeline_runs and
step_traces tables.
