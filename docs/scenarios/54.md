# Scenario 54: Web Dockerfile with Node.js and Next.js

## Requirement

> The web Dockerfile SHALL be node based using next.js.

## Implementation

`web/Dockerfile` implements a **three-stage Node.js build** for the Next.js web
GUI:

### Stage 1 — Build (cross-platform)

```dockerfile
FROM --platform=$BUILDPLATFORM node:22-alpine AS builder
WORKDIR /app
COPY package.json package-lock.json ./
RUN npm ci --ignore-scripts
COPY . .
RUN npm run build
```

Uses `--platform=$BUILDPLATFORM` to allow cross-compilation without QEMU.
`npm run build` runs `next build`, producing a standalone Next.js output at
`.next/standalone/`.

### Stage 2 — Native modules (target platform)

```dockerfile
FROM node:22-alpine AS native-deps
RUN apk add --no-cache python3 make g++
COPY package.json package-lock.json ./
RUN npm ci
```

Compiles `better-sqlite3` (a native addon) for the target platform separately so
the binary matches the deployment architecture.

### Stage 3 — Runtime

```dockerfile
FROM node:22-alpine
ENV NODE_ENV=production
COPY --from=builder /app/.next/standalone ./
COPY --from=builder /app/.next/static ./.next/static
COPY --from=builder /app/db/migrations ./db/migrations
COPY --from=native-deps /app/node_modules/better-sqlite3 ./node_modules/better-sqlite3
RUN mkdir -p /app/data
EXPOSE 3000
CMD ["node", "server.js"]
```

Runs the Next.js standalone server via `node server.js` on port 3000.

### Key File

- `web/Dockerfile` — Node.js/Next.js web GUI Dockerfile
