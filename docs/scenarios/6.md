# Scenario 6: Docker-Enabled Pipeline Execution

> Pipelines SHALL be runnable in any docker enabled environment

## Implementation

Thingfactory builds as a multi-stage Docker image and runs pipelines via a Node.js CLI entrypoint.

### Dockerfile

Two-stage build:
1. **Builder**: Node 22 Alpine + Gleam 1.13.0 — downloads deps, compiles to JavaScript target
2. **Runtime**: Node 22 Alpine — copies compiled JS, sets `ENTRYPOINT ["node", "bin/cli.mjs"]`

### CLI Entry Point (`bin/cli.mjs`)

Shim that imports the compiled Gleam `main()` and runs it. Enables `docker run thingfactory run basic`.

### Command Runner (`src/thingfactory/command_runner.gleam`)

- `run(program, args)` — executes shell commands via FFI (JS: `child_process.execFileSync`, Erlang: `os:cmd`)
- `step(program, args)` — factory that creates pipeline steps from shell commands
- Used by `gleam_build_pipeline()` and `dogfood_pipeline()` for real command execution
