# Scenario 8: Loop and Re-entrant Pipelines

> Pipelines SHALL be able to loop, or be re-entrant

## Implementation

Steps support three loop strategies configured via `pipeline.add_step_with_loop(name, fn, loop)`:

### Loop Types (`src/thingfactory/types.gleam:165-172`)

- `FixedCount(count)` — repeat step N times
- `RetryOnFailure(max_attempts)` — retry on error up to N attempts, stop on success
- `UntilSuccess(max_attempts)` — keep trying until success or exhaustion

### Executor Support (`src/thingfactory/executor.gleam:322-464`)

- `run_loop()` dispatches to `run_fixed_count`, `run_retry_on_failure`, or `run_until_success`
- Each iteration creates a distinct `StepTrace` with `[iteration:N]` suffix for observability
- Loop traces are accumulated and merged into the execution result
