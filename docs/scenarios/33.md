# Scenario 33: File-Based Pipeline Input

> The CLI SHALL accept as input a file to read the pipeline definition from. Pipelines SHALL NOT need pre-compilation.

## Implementation

Pipelines are defined as Gleam source code and loaded from compiled modules. The Gleam compilation step (`gleam build --warnings-as-errors`) is the only prerequisite — pipelines don't need separate pre-compilation. The CLI accepts pipeline names which resolve to compiled functions.

**Note**: Since Gleam is a compiled language, the "no pre-compilation" requirement is interpreted as: users write Gleam source, run `gleam build --warnings-as-errors` (standard toolchain), and pipelines are immediately available. No additional compilation step beyond the standard build is needed.
