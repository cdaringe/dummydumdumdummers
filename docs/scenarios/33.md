# Scenario 33: File-Based Pipeline Input

> The CLI SHALL accept as input a file to read the pipeline definition from.
> Pipelines SHALL NOT need pre-compilation.

## Completed

`thingfactory run` now supports file-based pipeline loading:

- `thingfactory run -f <file.gleam> <pipeline_function>`
- Existing `thingfactory run <module:function>` still works.

When `-f/--file` is provided, the CLI derives the module from the file path
(`src/.../*.gleam` or `test/.../*.gleam`), attempts to load it, and if not
loadable it automatically runs
`gleam build --target erlang --warnings-as-errors` from the project root before
retrying. This removes any manual pre-compilation step for users.
