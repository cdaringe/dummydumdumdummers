# Scenario 36: Compact and Verbose CLI Progress

> The CLI SHALL show compact and verbose modes of progress along a pipeline.

## Completed Behavior

- Compact mode (`run -c`) stays concise: per-step status lines only.
- Verbose mode (`run`) now prints step output payloads for each completed step in addition to step start/finish status.
- Step output is propagated from executors into CLI progress events, so command step output and other returned step output are visible during the run.

## Evidence

| Artifact | Proof |
|---|---|
| `src/thingfactory/types.gleam` | `StepEvent.StepFinished` now includes `output: String`. |
| `src/thingfactory/executor.gleam` | Sequential execution populates `StepFinished(..., output)` from step result output. |
| `src/thingfactory/parallel_executor.gleam` | DAG execution populates `StepFinished(..., output)` from step result output. |
| `src/thingfactory/cli.gleam` | `verbose_progress` prints non-empty `StepFinished.output`; `compact_progress` ignores it. |
| `test/cli_test.gleam` | CLI run/execute tests pass with compact + verbose mode behavior still intact. |
