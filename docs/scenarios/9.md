# Scenario 9: Arbitrary Compute Tasks

> Pipelines SHALL support running arbitrary compute tasks

## Implementation

### Command Runner (`src/thingfactory/command_runner.gleam`)

- `run(program, args)` — execute any shell command via FFI, returning
  `CommandOutput(exit_code, stdout, stderr)`
- `step(program, args)` — factory creating pipeline-compatible step functions
  from shell commands
- FFI backends: JavaScript (`child_process.execFileSync`) and Erlang (`os:cmd`)

### Pipeline Steps as Functions

Any `fn(Context, Dynamic) -> Result(Dynamic, StepError)` can be a pipeline step.
This includes:

- Shell commands via `command_runner.step()`
- Kubernetes Jobs via `kubernetes_runner.step()`
- Custom Gleam logic
- HTTP calls, file operations, etc.
