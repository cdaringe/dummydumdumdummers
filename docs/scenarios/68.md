# Scenario 68: Allow Listing of Permissible Execution Models

**Specification:** The system SHALL support allow listing of permissible
execution models.

## Implementation

### `web/lib/config.ts` — `allowedExecutors` field

`ServiceConfig` gains an `allowedExecutors: ExecutorKind[]` field driven by
`THINGFACTORY_ALLOWED_EXECUTORS` (comma-separated, defaults to `"local"`):

```typescript
const allowedExecutorsRaw = process.env.THINGFACTORY_ALLOWED_EXECUTORS ??
  "local";
const allowedExecutors = allowedExecutorsRaw
  .split(",")
  .map((s) => s.trim())
  .filter((s): s is ExecutorKind => s === "local" || s === "docker");
if (allowedExecutors.length === 0) allowedExecutors.push("local");
```

Unknown kinds are silently dropped; the fallback ensures at least `"local"` is
always present.

### `web/lib/trigger-pipeline.ts` — Enforcement

Before creating a run, `triggerPipeline` checks the pipeline's executor kind
against the allow list:

```typescript
if (!config.allowedExecutors.includes(executor.kind)) {
  throw new Error(
    `Executor kind "${executor.kind}" is not permitted. Allowed: ${
      config.allowedExecutors.join(", ")
    }`,
  );
}
```

The calling route (`POST /api/pipelines/[name]/[version]/trigger`) returns
**HTTP 403** when this check fails:

```typescript
try {
  runId = await triggerPipeline(pipelineId, "manual");
} catch (err) {
  const msg = err instanceof Error ? err.message : String(err);
  return NextResponse.json({ error: msg }, { status: 403 });
}
```

Webhook routes (`/api/webhooks/github`, `/api/webhooks/gitea`) similarly catch
and surface these errors.

### `docker-compose.yml` — Configurable allow list

```yaml
environment:
  - THINGFACTORY_ALLOWED_EXECUTORS=${THINGFACTORY_ALLOWED_EXECUTORS:-local}
```

Operators set `THINGFACTORY_ALLOWED_EXECUTORS=local,docker` to enable both kinds
in production.

## Variables Reference

| Variable                         | Default | Description                                      |
| -------------------------------- | ------- | ------------------------------------------------ |
| `THINGFACTORY_ALLOWED_EXECUTORS` | `local` | Comma-separated list of permitted executor kinds |

## Verification

- `config.ts` parses and filters the env var; ensures non-empty list.
- `trigger-pipeline.ts` throws on disallowed kind; callers return 403.
- `docker-compose.yml` wires the env var through to the container.
