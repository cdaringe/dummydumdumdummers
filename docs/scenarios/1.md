# Scenario 1: README Comprehension

> A user SHALL be able to visit the README and understand what the project
> offers: a CLI, a web GUI, a pipeline runner & orchestrator.

## Implementation

The `README.md` opens with a one-line description and immediately lists all
three components:

```
**CLI** | **Web GUI** | **Pipeline Runner & Orchestrator**
```

Sections cover: What It Does, Quick Start (install, run, define, parallel, test,
Docker), Features, Documentation links, Examples, and Development.

### Evidence

| Artifact           | Description                             |
| ------------------ | --------------------------------------- |
| `README.md:1-5`    | Title + component listing               |
| `README.md:8-15`   | "What It Does" feature summary          |
| `README.md:83-96`  | Features list covering all capabilities |
| `README.md:98-107` | Documentation table linking 6 guides    |
