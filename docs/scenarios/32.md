# Scenario 32: No Embedded Examples in Binary

> The CLI SHALL NOT embed examples pipelines in the binary or compiled artifact.

## Implementation

`thingfactory/cli` no longer imports or dispatches to `thingfactory/examples`.
`run` now accepts a runtime pipeline reference (`module:function`) and loads it
through FFI at execution time. This removes all hardcoded example pipeline
mappings from the CLI artifact itself.

### Evidence

| Artifact                                       | Description                                                                                                                         |
| ---------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------- |
| `src/thingfactory/cli.gleam`                   | `execute_pipeline()` parses `module:function` and calls `load_pipeline()`. No `examples` import or name/number switch table exists. |
| `src/thingfactory/thingfactory_erlang_cli.erl` | Implements `load_pipeline/2` with runtime `apply(Module, Function, [])`.                                                            |
| `src/thingfactory/cli_ffi.mjs`                 | JS target explicitly reports dynamic loading is Erlang-target runtime behavior (no embedded pipeline table fallback).               |
| `test/cli_test.gleam`                          | Verifies runtime ref execution (`thingfactory@examples:basic_pipeline`) and invalid/unknown ref failures.                           |
