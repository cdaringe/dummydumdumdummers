# Scenario 28: No String Typing

> The pipeline tasks SHALL NOT be stringly typed or referenced. Stringly typing in general is to be minimized.

## Implementation

Gleam's type system eliminates stringly-typed references:

- **Pipeline IDs** — `PipelineId(name, version)` record type, not bare strings
- **Step errors** — `StepError` sum type: `StepFailure | StepTimeout | ArtifactNotFound`
- **Pipeline errors** — `PipelineError` sum type: `StepError | LoadError | ValidationError`
- **Schedules** — `Schedule` sum type with 6 variants (not cron strings as primary)
- **Triggers** — `Trigger` and `WebhookMatcher` sum types with typed matchers
- **Loops** — `Loop` sum type: `FixedCount | RetryOnFailure | UntilSuccess`
- **Context** — typed `Dict(String, Dynamic)` with `get_dep()` returning `Result`

Step names are strings (necessary for user-defined names) but all structural references are typed.
