# Scenario 28: No String Typing

> The pipeline tasks SHALL NOT be stringly typed or referenced. Stringly typing
> in general is to be minimized.

## Implementation

Gleam's type system eliminates stringly-typed references:

- **Pipeline IDs** — `PipelineId(name, version)` record type, not bare strings
- **Step errors** — `StepError` sum type:
  `StepFailure | StepTimeout | ArtifactNotFound`
- **Pipeline errors** — `PipelineError` sum type:
  `StepError | LoadError | ValidationError`
- **Schedules** — `Schedule` sum type with 6 variants (not cron strings as
  primary)
- **Triggers** — `Trigger` and `WebhookMatcher` sum types with typed matchers
- **Loops** — `Loop` sum type: `FixedCount | RetryOnFailure | UntilSuccess`
- **Context** — typed `Dict(String, Dynamic)` with `get_dep()` returning
  `Result`
- **Step dependencies** — `StepRef` type; `add_step_with_deps` accepts
  `List(StepRef)` not `List(String)`

### Step Dependency References

Step-to-step dependencies use the `pipeline.StepRef` type, created via
`pipeline.step_ref(name)`. References are typed tokens rather than bare strings:

```gleam
pipeline.new("parallel_build", "1.0.0")
|> pipeline.add_step("checkout", checkout_fn)
|> pipeline.add_step_with_deps("typescript", ts_fn, [pipeline.step_ref("checkout")])
|> pipeline.add_step_with_deps("go", go_fn, [pipeline.step_ref("checkout")])
|> pipeline.add_step_with_deps("verify", verify_fn, [
    pipeline.step_ref("typescript"),
    pipeline.step_ref("go"),
  ])
```
