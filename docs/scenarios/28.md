# Scenario 28: No String Typing

> The pipeline tasks SHALL NOT be stringly typed or referenced. Stringly typing
> in general is to be minimized.

## Implementation

Gleam's type system eliminates stringly-typed references throughout the pipeline
system. The key mechanism is the `Pipeline(id, output)` phantom type, where `id`
is the step-identifier type â€” users supply an enum rather than bare strings.

### Typed Step IDs in `pipeline.typed()`

`pipeline.typed(name, version, to_string)` accepts any type as the step
identifier. Step references in `depends_on` lists carry the same type, so the
Gleam compiler rejects unknown or misspelled step names at build time:

```gleam
type BuildStep { Checkout Lint Test Build }

pipeline.typed("parallel_build", "1.0.0", step_to_string)
|> pipeline.add_step_with_deps(Checkout, checkout_fn, [])
|> pipeline.add_step_with_deps(Lint,     lint_fn,     [Checkout])
|> pipeline.add_step_with_deps(Test,     test_fn,     [Checkout])
|> pipeline.add_step_with_deps(Build,    build_fn,    [Lint, Test])
```

`depends_on` holds `List(id)` â€” the same type as `name`. No string coercion is
possible; the compiler enforces correctness.

### Flagship Pipeline â€” `build.gleam`

The canonical self-build pipeline (`src/thingfactory/build.gleam`) uses the
typed API with a 13-variant `BuildStep` enum:

```gleam
pub type BuildStep {
  GleamCheck  GleamFormat  WebInstall
  GleamTest   WebLint
  GleamBuildErl  GleamBuildJs  WebBuild
  CliShipment    DockerBuildCli  DockerBuildWeb
  HexPublish
  SemanticRelease
}

pub fn build() -> pipeline.Pipeline(BuildStep, Dynamic) {
  pipeline.typed("thingfactory-build", "1.0.0", build_step_to_string)
  |> pipeline.add_step_with_deps(GleamCheck, ..., [])
  |> pipeline.add_step_with_deps(GleamTest,  ..., [GleamCheck, GleamFormat])
  ...
}
```

Mistyping a step name (e.g. `GleamChekc`) is a compile error, not a runtime
failure.

### Other typed guarantees across the system

| Construct       | Type                                                                   |
| --------------- | ---------------------------------------------------------------------- |
| Pipeline IDs    | `PipelineId(name, version)` record                                     |
| Step errors     | `StepError` sum type: `StepFailure \| StepTimeout \| ArtifactNotFound` |
| Pipeline errors | `PipelineError` sum type                                               |
| Schedules       | `Schedule` sum type with 6 variants                                    |
| Triggers        | `Trigger` / `WebhookMatcher` sum types                                 |
| Loops           | `Loop` sum type: `FixedCount \| RetryOnFailure \| UntilSuccess`        |
