# Scenario 28: No String Typing

> The pipeline tasks SHALL NOT be stringly typed or referenced. Stringly typing
> in general is to be minimized.

## Implementation

Gleam's type system eliminates stringly-typed step references. The
`pipeline.new()` function accepts **any type** as the step identifier — users
supply an enum instead of bare strings. No manual `to_string` converter is
required.

### Single unified API — `pipeline.new()`

`pipeline.new(name, version)` works for both string and typed enum step IDs. The
`id_to_string` conversion is handled automatically via
`pipeline.step_id_to_string`:

- **String IDs** — used verbatim (`"my_step"` → `"my_step"`)
- **Enum IDs** — serialised with `string.inspect` (`GleamCheck` →
  `"GleamCheck"`, `Worker(3)` → `"Worker(3)"`)

No separate `pipeline.typed()` function exists; no manual converter is needed.

```gleam
type BuildStep { Checkout Lint Test Build }

pipeline.new("parallel_build", "1.0.0")
|> pipeline.add_step_with_deps(Checkout, checkout_fn, [])
|> pipeline.add_step_with_deps(Lint,     lint_fn,     [Checkout])
|> pipeline.add_step_with_deps(Test,     test_fn,     [Checkout])
|> pipeline.add_step_with_deps(Build,    build_fn,    [Lint, Test])
```

`depends_on` holds `List(id)` — the same type as the step name. No string
coercion is possible; the compiler enforces correctness.

### Flagship Pipeline — `build.gleam`

The canonical self-build pipeline (`src/thingfactory/build.gleam`) uses the
typed API with a 13-variant `BuildStep` enum and no `to_string` converter:

```gleam
pub type BuildStep {
  GleamCheck  GleamFormat  WebInstall
  GleamTest   WebLint
  GleamBuildErl  GleamBuildJs  WebBuild
  CliShipment    DockerBuildCli  DockerBuildWeb
  HexPublish
  SemanticRelease
}

pub fn build() -> pipeline.Pipeline(BuildStep, Dynamic) {
  pipeline.new("thingfactory-build", "1.0.0")
  |> pipeline.add_step_with_deps(GleamCheck, ..., [])
  |> pipeline.add_step_with_deps(GleamTest,  ..., [GleamCheck, GleamFormat])
  ...
}
```

Mistyping a step name (e.g. `GleamChekc`) is a compile error, not a runtime
failure.

### `step_id_to_string` — automatic serialisation

`pipeline.step_id_to_string` converts any step ID to a String:

```gleam
// In src/thingfactory/pipeline.gleam
pub fn step_id_to_string(id: a) -> String {
  let inspected = string.inspect(id)
  case string.starts_with(inspected, "\""), string.ends_with(inspected, "\"") {
    True, True -> string.slice(inspected, 1, string.length(inspected) - 2)
    _, _ -> inspected
  }
}
```

- `"step_name"` (String) → `"step_name"` (quotes stripped from inspect output)
- `GleamCheck` (enum) → `"GleamCheck"`
- `Worker(3)` (enum with data) → `"Worker(3)"`

### Other typed guarantees across the system

| Construct       | Type                                                                   |
| --------------- | ---------------------------------------------------------------------- |
| Pipeline IDs    | `PipelineId(name, version)` record                                     |
| Step errors     | `StepError` sum type: `StepFailure \| StepTimeout \| ArtifactNotFound` |
| Pipeline errors | `PipelineError` sum type                                               |
| Schedules       | `Schedule` sum type with 6 variants                                    |
| Triggers        | `Trigger` / `WebhookMatcher` sum types                                 |
| Loops           | `Loop` sum type: `FixedCount \| RetryOnFailure \| UntilSuccess`        |
