# Scenario 7: Artifact Sharing

> Pipelines SHALL have the ability to share artifacts

## Implementation

Inter-step data sharing uses a `Dict(String, Dynamic)` artifact store embedded
in `Context`. Steps write artifacts that downstream steps read.

### Artifact Store (`src/thingfactory/artifact_store.gleam`)

- `write(store, key, value)` — store a value by key (pure functional update)
- `read(store, key)` — retrieve value or `ArtifactNotFound` error
- `has_key(store, key)` — check existence
- `keys(store)` — list all artifact keys

### Context Integration (`src/thingfactory/types.gleam`)

`Context.artifact_store` carries artifacts through the pipeline. Steps using
`add_step_with_ctx` can update the context to write artifacts. The
`ExecutionResult.artifacts` field exposes final artifacts after execution.
