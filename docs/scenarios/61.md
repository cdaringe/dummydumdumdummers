# Scenario 61 — Docker Deployment Documentation & Builds-to-Jobs Guide

## Requirement

> The project SHALL host docker deployment documentation and instructions on how
> to get builds into jobs into the system.

## Implementation

### Primary Document

`docs/BUILDS_TO_JOBS.md` — end-to-end guide covering the complete workflow from
Docker deployment to VCS push events creating pipeline runs.

Sections:

1. **Overview** — ASCII diagram showing the GitHub/Gitea → webhook → pipeline
   run flow
2. **Step 1: Deploy with Docker** — single-container and docker-compose setups,
   public exposure via reverse proxy or ngrok
3. **Step 2: Define Pipelines with Triggers** — Gleam code example using
   `trigger.BranchUpdate("main")`
4. **Step 3: Register Repository Connections** — step-by-step for GitHub and
   Gitea on the `/integrations` UI page
5. **Step 4: Configure Webhooks in Your VCS** — webhook setup in GitHub and
   Gitea pointing to `/api/webhooks/github` and `/api/webhooks/gitea`
6. **Step 5: Verify the Flow** — `curl` payloads for synthetic testing, expected
   JSON response shape
7. **Troubleshooting** — table of common failure modes with fixes

### Supporting Documents (existing)

- `DOCKER.md` — Docker image details, docker-compose reference, CLI usage
- `docs/HOSTING_SERVICE.md` — production reverse-proxy, Kubernetes, config
  variables
- `docs/scenarios/56.md` — GitHub integrations UI (scenario 56)
- `docs/scenarios/57.md` — Gitea integrations UI (scenario 57)
- `docs/scenarios/59.md` — Webhook receivers that activate triggers
  (scenario 59)

### Webhook Endpoints (implemented in scenario 59)

- `POST /api/webhooks/github` — `web/app/api/webhooks/github/route.ts`
- `POST /api/webhooks/gitea` — `web/app/api/webhooks/gitea/route.ts`

Both endpoints accept push payloads, match against registered connections, and
call `triggerPipeline()` from `web/lib/trigger-pipeline.ts`.
