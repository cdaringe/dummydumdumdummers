# Scenario 4: Type-Safe Gleam Pipeline Definitions

> Pipelines SHALL be buildable via typesafe gleam code

## Implementation

Pipelines are defined using a type-safe builder API in Gleam. The
`Pipeline(id, output)` type is parameterised on both the step identifier type
and the final output type, enforced at compile time.

### Builder API (`src/thingfactory/pipeline.gleam`)

One constructor is provided:

- `pipeline.new(name, version)` — creates a pipeline; the step ID type `id` is
  inferred from the first step added, so both `String` IDs and custom enum types
  work without any extra constructor.

Common builder methods:

- `pipeline.add_step(id, fn)` — append a step
- `pipeline.add_step_with_deps(id, fn, deps)` — step with DAG dependencies
- `pipeline.add_step_with_loop(id, fn, loop)` — step with loop configuration
- `pipeline.add_step_with_ctx(id, fn)` — step that returns an updated Context
- `pipeline.with_timeout()`, `with_schedule()`, `with_trigger()`,
  `with_secrets()` — pipeline-level settings

Each step function has a typed signature:
`fn(Context, Dynamic) -> Result(Dynamic, StepError)`. The Gleam compiler
enforces these signatures at call sites. Intermediate types are erased to
`Dynamic` between steps (documented trade-off).

### Typed Step IDs (compile-time safety)

```gleam
type BuildStep { Checkout Lint Test Build Package }

pipeline.new("ci", "1.0.0")
|> pipeline.add_step_with_deps(Checkout, checkout_fn, [])
|> pipeline.add_step_with_deps(Lint, lint_fn, [Checkout])
|> pipeline.add_step_with_deps(Build, build_fn, [Lint])
```

Misspelled or missing step IDs are caught by the Gleam compiler before runtime.

### Core Types (`src/thingfactory/types.gleam`)

- `PipelineId(name, version)` — unique pipeline identity
- `StepError` — `StepFailure | StepTimeout | ArtifactNotFound` (no silent
  failures)
- `PipelineError` — `StepError | LoadError | ValidationError`
- `Context` — artifact store, message bus, dependency bindings, secrets
- `ExecutionResult(a)` — result + trace + artifacts

### Key Design Decisions

- Pipeline type is public (not opaque) to enable test mocking
- Steps stored internally as type-erased `fn(Context, Dynamic)` for flexibility
- `pipeline.compile()` converts `Pipeline(id, output)` to
  `Pipeline(String, output)` before execution
- Gleam compiler catches type mismatches at compile time, not runtime
