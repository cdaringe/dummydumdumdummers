# Scenario 22: Real Sample Build Pipelines

> The source code should have a large set of sample pipelines, demonstrating: 1. compilation of libraries for typescript, go, & gleam. They should be REAL, not mocked. 2. demonstration of defining a custom Docker image to run a pipeline in.

## Implementation

Real build pipelines use `command_runner.step_in_dir()` to execute actual build tools against real example projects in `examples/`. No hardcoded return values — every step runs actual toolchain commands.

### Real Example Projects

| Project | Location | Files |
|---|---|---|
| TypeScript library | `examples/typescript-lib/` | `package.json`, `tsconfig.json`, `src/index.ts` |
| Go library | `examples/go-lib/` | `go.mod`, `math.go`, `math_test.go` |
| Gleam (this project) | project root | `gleam.toml`, `src/`, `test/` |

### Real Build Pipelines

**`typescript_build_pipeline()`** — runs actual npm/tsc commands against `examples/typescript-lib/`:
1. `npm install` — installs TypeScript 5.x dependency
2. `npm run lint` — type-checks via `tsc --noEmit`
3. `npm run build` — compiles TypeScript source to `dist/`
4. `npm run test` — runs compiled output

**`go_build_pipeline()`** — runs actual go toolchain commands against `examples/go-lib/`:
1. `go mod download` — downloads module dependencies
2. `go test -v ./...` — runs all unit tests (4 tests)
3. `go build ./...` — compiles the library package
4. `go vet ./...` — static analysis

**`gleam_build_pipeline()`** — runs actual gleam commands against the project root:
1. `gleam check` — type checking
2. `gleam test` — unit tests
3. `gleam format --check` — format check
4. `gleam build --warnings-as-errors --target javascript`
5. `gleam build --warnings-as-errors --target erlang`

### Custom Docker Image

`Dockerfile` defines a custom multi-stage image (Node 22 Alpine + Gleam 1.13.0).
